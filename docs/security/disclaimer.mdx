Trading perpetual futures involves substantial risk of loss. Live orders, rebalances and position closes on Pear are **irreversible** and move real funds. Only enable trading with capital you can afford to lose.
</Warning>

## No financial advice

Pear Pro MCP is software tooling, not investment advice. The analytics it computes (correlation, beta, z-scores, signals) are statistical descriptions of historical data, not predictions or recommendations. A `signal` of `extended_long_spread` is a label, not a suggestion to trade. You are solely responsible for every order you place.

## Software is provided "as is"

This project is **alpha (`0.1.0`)** and provided under the MIT license without warranty of any kind. Expect schema-level changes between minor versions until `1.0`. Bugs, API changes on Pear's side, network failures or misconfiguration can all lead to unexpected behavior — including failed or unintended orders.

## You control the safety switches

The server defaults to read-only and requires explicit `confirm: true` per trade for a reason. By setting `PEAR_READ_ONLY=false`, passing `confirm: true`, or enabling `PEAR_AUTO_REBALANCE`, you accept full responsibility for the resulting trades. The defenses described in the [threat model](/security/threat-model) reduce *accidental* fund movement; they do not guarantee profit or prevent loss.

## Third-party services

This server talks to the Pear Protocol API and, transitively, to the underlying Hyperliquid / Lighter venues. Their availability, correctness, fees and terms are outside this project's control. Review Pear's own terms and documentation at [docs.pearprotocol.io](https://docs.pearprotocol.io).

## Your keys, your funds

You are responsible for safeguarding your wallet, API keys and access tokens. Anyone with your `PEAR_API_KEY` and a trading-enabled server can move your funds. Treat these as production secrets.
