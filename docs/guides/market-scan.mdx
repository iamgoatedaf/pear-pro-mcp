The market-data tools need no API key and never touch funds. They're the fastest way to confirm the server works and to feed an agent real market context.

## The four read tools

| Tool | Returns |
| --- | --- |
| `pear_health` | Engine availability |
| `pear_get_markets` | Markets with OI, volume, ratio, weighted ratio, net funding — filter & sort |
| `pear_active_markets` | Active pairs + gainers/losers/highlighted/watchlist |
| `pear_trade_ideas` | `active` / `watchlist` / `ai-picks` baskets |

## 1. Confirm the engine is up

> Call `pear_health`.

A successful response means the server reached `PEAR_API_BASE_URL`. If it fails, check the base URL and your network.

## 2. Browse and sort markets

`pear_get_markets` supports server-side filtering and sorting so the agent doesn't have to page through everything:

> Call `pear_get_markets` with `sort: "volume:desc"`, `pageSize: 10`. Then call it again with `netFunding: "positive"` and tell me which pairs pay to hold.

Useful parameters:

- `searchText` — free-text match
- `minVolume` — drop illiquid pairs
- `netFunding` — filter by funding sign
- `sort` — e.g. `"volume:desc"`
- `page` / `pageSize` — pagination (max 100/page)

## 3. Find today's movers

> Call `pear_active_markets` and summarize top gainers and losers.

This returns actively traded pairs plus the platform's gainers, losers, highlighted pairs and the user watchlist in one shot.

## 4. Surface trade ideas

> Call `pear_trade_ideas` with `category: "ai-picks"`.

Three categories: `active` (platform activity), `watchlist` (yours), and `ai-picks` (Pear's statistical-arbitrage engine). The server filters the basket list by category for you.

## Put it together

A typical agent prompt:

> Survey Pear: check health, pull the top 10 markets by volume, list today's biggest movers, and show me the AI-pick baskets. Then suggest one pair worth analyzing further.

From here, take a candidate pair into [pair analytics](/guides/pair-analytics) to quantify the entry.
