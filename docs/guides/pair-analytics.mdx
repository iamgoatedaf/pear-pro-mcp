`pear_pair_analytics` is the one read tool that needs **no API at all**. You pass two aligned price series and it returns the same stats Pear's "Agent Pair" reasons over, computed locally.

## How to read it

<AccordionGroup>
  <Accordion title="correlationOfReturns">
    Correlation of the two legs' log returns. High positive correlation is what makes a pair mean-revert rather than wander apart. Low correlation = a weak pair.
  </Accordion>
  <Accordion title="beta (hedge ratio)">
    OLS hedge ratio of A on B (`cov(a,b)/var(b)`). This is roughly how much of B you short per unit of A to make the pair market-neutral. Use it to size the legs.
  </Accordion>
  <Accordion title="zScore">
    Z-score of the latest A/B spread vs the trailing window. `|z| â‰Ą 2` is a classic mean-reversion entry. Positive = spread stretched high (A rich vs B); negative = spread stretched low.
  </Accordion>
  <Accordion title="rollingZScoreTail">
    The last 10 rolling z-scores, so you can see whether the spread is *entering* an extreme or already reverting.
  </Accordion>
  <Accordion title="signal">
    A convenience label: `extended_long_spread` when `z â‰Ą 2`, `extended_short_spread` when `z â‰¤ -2`, else `neutral`.
  </Accordion>
</AccordionGroup>

If the `signal` is `extended_short_spread` with strong correlation, the agent might propose going long the spread (long A / short B sized by `beta`), then hand off to [a guarded trade](/guides/guarded-trade).
