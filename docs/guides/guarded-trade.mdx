title: Open a guarded position

This guide opens an actual position, so it requires `PEAR_READ_ONLY=false`, a funded Pear account, and `PEAR_API_KEY` + `PEAR_ADDRESS`. Read [Safety](/get-started/safety) and the [Disclaimer](/security/disclaimer) first.

<Warning>
Live orders on Pear are **irreversible** and move real funds. Test the full flow in read-only mode first — you'll see exactly where the guard stops you.
</Warning>

## 1. Confirm the guard is working (read-only)

With `PEAR_READ_ONLY=true`, ask the agent to open a position:

> Open a 1x ETH/BTC pair, $100, long ETH short BTC, with `pear_open_position`.

It returns:

```
Server is in READ-ONLY mode (PEAR_READ_ONLY=true). Refusing to place/modify
orders. Set PEAR_READ_ONLY=false to enable trading.
```

Good — the gate works. Now set `PEAR_READ_ONLY=false` in your environment and restart the server.

## 2. The confirm gate

Call it again **without** `confirm`:

> Open the same ETH/BTC pair.

```
This action moves real funds. Re-call with confirm=true to execute.
```

This is the second layer. Nothing has traded yet.

## 3. Execute

Now pass `confirm: true`. Ask the agent to call `pear_open_position` with:

```json
{
  "usdValue": 100,
  "leverage": 1,
  "slippage": 0.01,
  "executionType": "MARKET",
  "longAssets": [{ "asset": "ETH" }],
  "shortAssets": [{ "asset": "BTC" }],
  "confirm": true
}
```

The order goes to Pear and you get the created position back. Because no weights were given, the legs split evenly.

## 4. Add risk management

Attach TP/SL — for a basket, `WEIGHTED_RATIO` thresholds make the most sense:

> Call `pear_set_risk` on that position with a stop-loss of 5% and a trailing take-profit, `confirm: true`.

TP/SL `type` can be `PERCENTAGE`, `DOLLAR`, `POSITION_VALUE`, `PRICE`, `PRICE_RATIO` or `WEIGHTED_RATIO`, and supports trailing via `isTrailing` + `trailingDeltaValue` + `trailingActivationValue`.

## 5. Manage and close

| Action | Tool |
| --- | --- |
| Resize by % | `pear_adjust_position` (`adjustmentType: REDUCE`/`INCREASE`, `adjustmentSize` 1–100) |
| Rebalance legs to target | `pear_rebalance_execute` (preview with `pear_rebalance_plan` first) |
| Close one | `pear_close_position` (`MARKET`/`TWAP`/`TRIGGER`) |
| Close everything | `pear_close_all_positions` |
| Cancel a resting order | `pear_cancel_order` |

Every one of these is guarded: `PEAR_READ_ONLY=false` **and** `confirm: true`.

## Advanced order types

`pear_open_position` supports more than `MARKET`:

- **TRIGGER** — set `triggerType` (`PRICE`, `PRICE_RATIO`, `WEIGHTED_RATIO`, `BTC_DOM`, …), `triggerValue` and `direction` (`MORE_THAN`/`LESS_THAN`).
- **TWAP** — set `twapDuration` (minutes) and optional `twapIntervalSeconds`.
- **LADDER** — scale in across levels.

See the full schema in [Tool reference → Trading](/tools/trading).
