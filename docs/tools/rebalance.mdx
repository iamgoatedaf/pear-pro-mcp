---
title: Rebalance preview tools
description: "Two non-mutating tools that compute what a rebalance would do — safe to poll."
---

Both tools hit the **non-mutating** plan endpoint (`POST /positions/{id}/rebalance/plan`), so they never place an order. They're safe to call anytime, including on a schedule. To actually rebalance, use the guarded [`pear_rebalance_execute`](/tools/trading#pear-rebalance-execute).

## `pear_rebalance_plan`

Compute weight deltas and what *would* be traded to reach target weights, without placing any orders.

<ParamField path="positionId" type="string" required>Position identifier.</ParamField>
<ParamField path="targetWeights" type="object">
  Optional per-asset target weights (decimals summing to 1.0 per side). Omit to use the position's stored targets.
</ParamField>

`POST /positions/{positionId}/rebalance/plan` with `{ targetWeights }`

Returns a plan with, per asset: `coin`, `side`, `currentWeight`, `targetWeight`, `currentValue`, `targetValue`, `deltaValue`, `currentSize`, `newSize`, `deltaSize`, and `skipped` (+ `skipReason`), plus a top-level `canExecute`.

> `pear_rebalance_plan` for position `abc123`. What would trade to get back to target?

---

## `pear_rebalance_check`

Analyze how far a position's leg weights have drifted from target. Returns max drift %, per-asset trim/add actions and a recommendation.

<ParamField path="positionId" type="string" required>Position identifier.</ParamField>
<ParamField path="driftThreshold" type="number" default="0.05">
  Drift fraction that triggers a recommendation (`0.05` = 5%). Range 0–1.
</ParamField>
<ParamField path="targetWeights" type="object">
  Optional target overrides; omit to use the position's stored targets.
</ParamField>

Internally fetches the plan, then derives a **drift report**:

```json
{
  "positionId": "abc123",
  "maxDriftPct": 7.2,
  "rebalanceRecommended": true,
  "threshold": 0.05,
  "canExecute": true,
  "assets": [
    {
      "coin": "ETH", "side": "long",
      "currentWeightPct": 57.2, "targetWeightPct": 50,
      "driftPct": 7.2, "deltaValueUsd": -120.5,
      "action": "trim", "skipped": false
    }
  ],
  "summary": "Rebalance recommended: max drift 7.2% (threshold 5%). Worst leg ETH 57.2%→50%."
}
```

`action` is `trim` (deltaValue < 0), `add` (> 0) or `hold`. `rebalanceRecommended` is true only when `maxDrift ≥ threshold` **and** `canExecute`. Math: [Internals → Rebalance & drift math](/internals/rebalance-math).

> `pear_rebalance_check` for `abc123` with `driftThreshold: 0.03`. Should I rebalance?
