title: Auth flow
description: "How PearClient logs in, attaches JWTs, and refreshes them automatically."

`PearClient` (`src/client.ts`) handles Pear's documented auth flow so tools don't have to. EIP-712 wallet signing is intentionally **out of scope** — agents/bots use a long-lived API key created once after a wallet login.

## Getting an API key (one-time, manual)

<Steps>
  <Step title="Wallet login">
    Authenticate in the Pear app, or `POST /auth/authenticate` with `method: "eip712"`.
  </Step>
  <Step title="Mint a key">
    `POST /api-keys` → store the returned `apiKey`.
  </Step>
  <Step title="Configure">
    Put it in the environment as `PEAR_API_KEY`, with `PEAR_ADDRESS` and `PEAR_CLIENT_ID`.
  </Step>
</Steps>

## How the client authenticates

```
1. POST /auth/authenticate  { method: "api_key", address, clientId, details: { apiKey } }
      → { accessToken, refreshToken }
2. Authorization: Bearer <accessToken>   on every authenticated call
3. POST /auth/refresh  { refreshToken }   when the access token goes stale
```

The client treats both `accessToken`/`refreshToken` and `access_token`/`refresh_token` response shapes as valid.

## Token lifecycle

- After login or refresh, the access token is marked stale **14 minutes** out (`expiresAt`).
- `ensureAuth()` runs before every authenticated request: if the token is still fresh it's reused; if a refresh token exists it tries `POST /auth/refresh`; otherwise it does a fresh `login()`.
- A bare `PEAR_ACCESS_TOKEN` (no API key) is trusted for ~10 minutes and cannot be refreshed — so it's only useful for quick tests.

## Automatic 401 retry

`request()` wraps every authenticated call: on a `401`, it clears the cached tokens, re-authenticates, and retries the request **once**. If it still fails, the `PearApiError` propagates and the tool returns `Pear API error 401 …`.

```ts
async request(method, endpoint, opts) {
  await this.ensureAuth();
  try {
    return await this.rawRequest(method, endpoint, opts);
  } catch (err) {
    if (err instanceof PearApiError && err.status === 401) {
      this.tokens = null;
      await this.ensureAuth();
      return await this.rawRequest(method, endpoint, opts);
    }
    throw err;
  }
}
```

## Unauthenticated reads

`get()` is a thin wrapper that calls the API with `auth: false` — no `Authorization` header. It backs the market-data tools (`/health`, `/markets`, `/markets/active`, `/markets/v2`, `/public-stats/address`), which is why they work with zero configuration.

<Note>
The agent never sees the API key or any JWT. Tokens live only inside the server process; the agent exchanges tool inputs and outputs over stdio.
</Note>
