---
title: Cursor
description: "Add Pear Pro MCP to Cursor as an MCP server."
---

Cursor reads MCP servers from a JSON file — globally at `~/.cursor/mcp.json` or per-project at `.cursor/mcp.json` in your repo root.

## 1. Build the server

```bash
cd ~/pear-pro-mcp && npm install && npm run build
```

## 2. Add the config

Create or edit `~/.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "pear-pro": {
      "command": "node",
      "args": ["/Users/you/pear-pro-mcp/dist/index.js"],
      "env": {
        "PEAR_API_BASE_URL": "https://hl-v2.pearprotocol.io",
        "PEAR_CLIENT_ID": "APITRADER",
        "PEAR_READ_ONLY": "true"
      }
    }
  }
}
```

Add `PEAR_ADDRESS` and `PEAR_API_KEY` to `env` if you want account/trading tools. Keep `PEAR_READ_ONLY` as `"true"` until you've tested.

## 3. Enable it

Open **Cursor Settings → MCP**. You should see `pear-pro` with a green dot once it connects. Toggle it on if needed.

## 4. Use it

In a chat, ask the agent to call a tool:

> Call `pear_health`, then list the top 5 markets by volume with `pear_get_markets`.

<Tip>
For development without a build step, point Cursor at `tsx`:

```json
{
  "mcpServers": {
    "pear-pro": {
      "command": "npx",
      "args": ["tsx", "/Users/you/pear-pro-mcp/src/index.ts"],
      "env": { "PEAR_READ_ONLY": "true" }
    }
  }
}
```
</Tip>

<Note>
Changed the config or rebuilt? Reload the MCP server from **Cursor Settings → MCP** (or restart Cursor) so it picks up the new process.
</Note>
