Pear Pro MCP is configured entirely through environment variables, loaded from `.env` (via `dotenv`) or from the `env` block of your agent adapter. Nothing is required to run in read-only market-data mode.

## API connection

<ParamField path="PEAR_API_BASE_URL" type="string" default="https://hl-v2.pearprotocol.io">
  Base URL of the Pear engine to drive. `https://hl-v2.pearprotocol.io` is the documented Hyperliquid v2 engine. For the **Pear Pro x Lighter** engine, point this at the Lighter base URL (ask the Pear team). Trailing slashes are stripped automatically.
</ParamField>

<ParamField path="PEAR_CLIENT_ID" type="string" default="APITRADER">
  The `clientId` sent on every auth request. Individual traders use `APITRADER`. Partners building products on Pear use their own client id (request one from Pear) so volume and fees are attributed to them.
</ParamField>

<ParamField path="PEAR_ADDRESS" type="string">
  Your wallet address — the trading account identity. Required for account and trading tools; unused by market-data tools.
</ParamField>

## Authentication

You need **one** of the following for authenticated (account / trading) tools. Market-data and local-quant tools work with neither.

<ParamField path="PEAR_API_KEY" type="string">
  A long-lived API key created via `POST /api-keys` after a wallet login. **Preferred for bots/agents** — the server uses it to auto-authenticate (`method: "api_key"`) and silently refreshes JWTs. See [Internals → Auth](/internals/auth).
</ParamField>

<ParamField path="PEAR_ACCESS_TOKEN" type="string">
  An already-issued JWT access token. Use this to skip the auth flow entirely (e.g. for a quick test). The server assumes ~10 minutes of life and re-authenticates on a 401 — but only if `PEAR_API_KEY` is also set, so a bare access token is short-lived.
</ParamField>

## Safety rails

<ParamField path="PEAR_READ_ONLY" type="boolean" default="true">
  When `true` (the default), the server refuses **every** order-placing / position-mutating tool and explains how to enable trading. Set to `false` to allow trading — and even then each mutating call still needs `confirm: true`. Recommended: keep `true` for demos and any agent you don't fully trust. See [Safety](/get-started/safety).
</ParamField>

## Background rebalance monitor

These only affect `npm run monitor` (the [rebalance monitor](/guides/rebalance-monitor)), not the MCP server itself.

<ParamField path="PEAR_POLL_INTERVAL_MS" type="number" default="300000">
  Poll interval in milliseconds (minimum 5000). `300000` = 5 minutes. Values below 5000 or non-numeric fall back to the default.
</ParamField>

<ParamField path="PEAR_DRIFT_THRESHOLD" type="number" default="0.05">
  Recommend a rebalance when any leg's weight drifts this far from target. `0.05` = 5%.
</ParamField>

<ParamField path="PEAR_POSITION_ID" type="string">
  Watch a single position. Leave empty to watch all open positions.
</ParamField>

<ParamField path="PEAR_AUTO_REBALANCE" type="boolean" default="false">
  If `true` **and** `PEAR_READ_ONLY=false`, the monitor executes rebalances on its own. If `false` (default), it only notifies and you approve via chat (`rebalance <id>`). With `PEAR_READ_ONLY=true` the monitor stays alert-only even if this is `true`.
</ParamField>

## Truthy values

Boolean variables (`PEAR_READ_ONLY`, `PEAR_AUTO_REBALANCE`) treat `1`, `true`, `yes`, `on` (case-insensitive) as true. Anything else is false.

## Minimal configs

<CodeGroup>
```bash Read-only (no auth)
PEAR_API_BASE_URL=https://hl-v2.pearprotocol.io
PEAR_READ_ONLY=true
```

```bash Account read access
PEAR_API_BASE_URL=https://hl-v2.pearprotocol.io
PEAR_CLIENT_ID=APITRADER
PEAR_ADDRESS=0xYourWalletAddress
PEAR_API_KEY=your_api_key
PEAR_READ_ONLY=true
```

```bash Trading enabled
PEAR_API_BASE_URL=https://hl-v2.pearprotocol.io
PEAR_CLIENT_ID=APITRADER
PEAR_ADDRESS=0xYourWalletAddress
PEAR_API_KEY=your_api_key
PEAR_READ_ONLY=false
```
</CodeGroup>

<Warning>
`PEAR_READ_ONLY=false` is the switch that lets the agent move real money. Pair it with `confirm: true` discipline and keep your API key out of the agent's reach. Read [Safety](/get-started/safety) first.
</Warning>
