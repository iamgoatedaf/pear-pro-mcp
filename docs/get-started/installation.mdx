## Requirements

- **Node.js ≥ 18** (the server uses native `fetch` and ES modules).
- **npm** (ships with Node).
- A terminal. Trading and account tools additionally need a Pear account + API key — see [Configuration](/get-started/configuration). Market data and analytics need nothing.

## Install

```bash
git clone https://github.com/iamgoatedaf/pear-pro-mcp ~/pear-pro-mcp
cd ~/pear-pro-mcp
npm install
npm run build
```

`npm run build` runs `tsc` and emits the runnable server to `dist/`. The MCP entrypoint is `dist/index.js`.

## Configure environment

```bash
cp .env.example .env
```

Open `.env` and, at minimum, leave the defaults to run in **read-only market-data mode**. To enable account/trading tools, fill in `PEAR_ADDRESS` and `PEAR_API_KEY`. Full field-by-field reference is in [Configuration](/get-started/configuration).

<Note>
The server reads `.env` automatically via `dotenv`. When you wire it into an agent, you can instead pass the same variables in the adapter's `env` block — see [Agent adapters](/adapters/overview).
</Note>

## Verify the install

Run the server directly. It speaks MCP over stdio, so you won't see a prompt — it logs a readiness line to **stderr** and waits:

```bash
npm start
# pear-pro-mcp ready | base=https://hl-v2.pearprotocol.io | clientId=APITRADER | mode=READ-ONLY
```

Press `Ctrl+C` to stop. If you see `mode=READ-ONLY`, the safety rails are on. `mode=TRADING ENABLED` means `PEAR_READ_ONLY=false`.

<Tip>
During development use `npm run dev` (runs `src/index.ts` via `tsx`, no build step) and `npm run typecheck` (`tsc --noEmit`) to validate types without emitting.
</Tip>

## npm scripts

| Script | Command | What it does |
| --- | --- | --- |
| `build` | `tsc` | Compile TypeScript to `dist/` |
| `start` | `node dist/index.js` | Run the built MCP server |
| `dev` | `tsx src/index.ts` | Run the server from source, no build |
| `monitor` | `node dist/monitor.js` | Run the background rebalance monitor |
| `monitor:dev` | `tsx src/monitor.ts` | Run the monitor from source |
| `typecheck` | `tsc --noEmit` | Type-check only |
